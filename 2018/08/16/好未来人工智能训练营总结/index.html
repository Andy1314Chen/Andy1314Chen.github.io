好未来人工智能训练营总结

发表于 2018-08-16 | 分类于 ML

两周时间，2018-07-30至2018-08-13，TAL AI LAB数据挖掘训练营，学习到了不少知识。两周的作业都是优，最后拿到了数据挖掘训练营优秀营员。

先放一个训练营论坛链接，https://future.ailab.100tal.com/camp/detail/3/forum.

第一周

一、 Week1学习目标：

  • 利用Python进行数据分析
  • 推荐系统基础、了解常见评估方法
  • 实现推荐算法(协同过滤、基于内容推荐等)
  • Exact KNN Search & Approximated KNN Search
  • 矩阵分解在推荐系统中的应用
  • (week1任务) 基于Movielens数据集进行电影推荐

二、 实现任务基本思路：

  1. 最基本的推荐方法，用户评分Top-K、评分人数Top-K、发行时间等进行推荐；
  2. 基于内容进行推荐，每部电影都有导演、主演、流派、标签等等属性特征，根据用户最近所看电影的特征向量，与数据库中所有电影进行相似度计算，推荐Top-K;人工检验推荐是否合理。
  3. 协同过滤推荐，包括User-based CF & Item-based CF；User-based利用相似用户通常会有相似品味，利用某用户对电影的评分向量进行相似度计算；Item-based则认为相似电影会有相似的评分，利用某电影的评分向量进行相似度计算；
  4. 评分矩阵进行矩阵分解，得到用户矩阵和电影矩阵，表达了不同用户和电影的隐含特征；基于此，NMF/SVD/SVD++可用于推荐；

三、 基本实现步骤：

1)利用Python对movielens进行数据探索,可视化分析；

3) 基于内容推荐算法；

在Day01中对数据分析，发现对每个movie共有1128个tags, 对每个tag共有10993个movies, 将这些movie的tag看作是该movie的属性特征，构建109931128的moviestags矩阵，计算相似度。这样不用我们对movie提取属性特征，再根据TFIDE进行向量化，十分方便。

具体过程如下：
    建立tfidf_matrix矩阵的行与movieId一一对应关系；
    获得与该movieId余弦相似度list, 第一个是movieId本身；
    对list按照相似度分数进行排序，第一个为1，第二个是最为相似的电影；
    取得list Top10(忽略第一个);
    返回Top 10的movieId.    


相似度计算方法考虑三种：
    欧式距离
    皮尔逊相关系数
    余弦相似度

使用余弦相似度计算时，采用sklearn库中sklearn.metrics.pairwise.linear_kernel进行近似可以加速计算。(http://scikit-learn.org/stable/modules/metrics.html#linear-kernel)

4) 协同过滤推荐算法；

  •  User-based CF
  •  Item-based CF
User-based CF与Item-based CF过程类似，具体步骤如下：
-    构建user-item评分矩阵；
-    划分训练集train set、测试集test set;
-    计算相似度(欧式距离、余弦相似度、Pearson相似度)
-    基于相似度矩阵进行推荐
-    评估推荐优劣(RMSE, MAE, Recall@K)

        # 表6 User-based CF不同相似度方法结果对比
            RMSE    MAE    Pecision@5    Recall@5
        欧式距离相似度    3.0027    2.7816    1.0    0.3034
        皮尔逊系数相似度    2.7449    2.5089    1.0    0.4118
        余弦相似度    2.7668    2.5309    1.0    0.4085

Train set 与 test set的比例7:3， 从RMSE、MAE、Precision@5、Recall@5几项评估指标看，欧式距离相似度都是较差的，皮尔逊系数和余弦相似度结果相差不大，皮尔逊系数相似度整体要好一些。

# 表7 Item-based CF不同相似度方法结果对比
    RMSE    MAE    Pecision@5    Recall@5
欧式距离相似度    3.5100    3.3105    1.0    0.0884
皮尔逊系数相似度    3.3133    3.112    1.0    0.1943
余弦相似度    3.3037    3.0926    1.0    0.2183

从Item-based CF结果看，整体要比User-based CF差一些，3种相似度计算方法中欧式距离相似度仍然是较差的，而皮尔逊系数相似度与余弦相似度类似，但这里余弦相似度要更好一些。

Collaborative Filtering协同过滤的改进算法，有Top-k collaborative、Bias-subtracted Collaborative Filtering、Top-k & Bias-subtracted Collaborative Filtering等。

从表8和表9结果来看，Top-k CF算法的精度要更高一些，其次是Top-k & Bias-subtracted CF混合算法，但此时模型复杂度上升了。Bias-subtracted CF与CF差不多，说明实际中用户对电影的评分大致还是区分度的。

5) 矩阵分解推荐算法；


图7 矩阵分解图示

简单地讲，每位user和每个item都有自己的特性，用矩阵分解可以从原来user-item评分矩阵中分解出user-feature矩阵、feature-item矩阵，对user来说，feature是偏好，对item来说，是特性。另外，矩阵分解还可以用来降低维度。
